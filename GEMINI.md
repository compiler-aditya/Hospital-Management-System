# Hospital Management System - Project Overview for Gemini

This document provides a comprehensive overview of the Hospital Management System, designed to serve as foundational context for the Gemini CLI agent when interacting with this codebase.

## Project Overview

The Hospital Management System is a comprehensive web-based application built with **Flask (Python)**, **SQLite** for the database, and **Bootstrap** for the frontend. It provides distinct functionalities for Admin, Doctor, and Patient roles, facilitating the management of hospital operations including appointments, patient records, doctor profiles, and treatment histories.

**Key Features:**
-   **Admin:** Dashboard, doctor/patient management, appointment oversight.
-   **Doctor:** Appointment management, patient history, availability setting, treatment notes.
-   **Patient:** Doctor search, appointment booking/management, treatment history, profile updates.

**Technology Stack:**
-   **Backend:** Flask 3.0.0, Flask-Login (Authentication), Flask-SQLAlchemy (ORM).
-   **Database:** SQLite.
-   **Frontend:** Jinja2 (templating), HTML5, CSS3, Bootstrap 5.3.

## Building and Running

To set up and run the Hospital Management System locally, follow these steps:

1.  **Create a virtual environment:**
    ```bash
    python3 -m venv venv
    source venv/bin/activate # On Windows: venv\Scripts\activate
    ```

2.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

3.  **Run the application:**
    ```bash
    python run.py
    ```

4.  **Access the application:**
    Open your web browser and navigate to: `http://127.0.0.1:5000`

The database will be automatically created on the first run, along with default admin credentials (`admin`/`admin123`) and pre-populated departments.

## Development Conventions

-   **Framework:** Flask is the core web framework.
-   **Templating:** Jinja2 is used for rendering HTML templates, organized by user roles (admin, doctor, patient) and common components.
-   **Database:** SQLite is used, with Flask-SQLAlchemy managing ORM operations defined in `app/models.py`.
-   **Styling:** Frontend styling is based on Bootstrap 5.3, supplemented by custom CSS in `app/static/css/main.css`.
-   **Structure:** The project follows a typical Flask application structure with a main `app` directory containing `__init__.py`, `models.py`, `routes.py`, `templates/`, and `static/`.
-   **Authentication & Authorization:** Flask-Login handles user authentication, and role-based access control is implemented throughout the application.
